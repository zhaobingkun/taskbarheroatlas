Build test template

  1. Record the current patch, stage and target activity.
  2. State the party, key gear assumptions and Rune goal.
  3. Change one meaningful input instead of replacing every slot.
  4. Keep the comparison only when the result repeats.

Why this page has no static best-build list

Build recommendations can become wrong after a patch, gear change or different stage condition. Atlas will publish named routes only with a version, role, target, inputs and observed result. That makes a smaller Build hub more useful than an impressive-looking but untestable ranking.
