SHORT ANSWER

There is no universal best Rune route. Pick the next bottleneck, take the branch that changes that result, and reassess after a meaningful gear, stage, or patch change.

Choose a rune goal before you choose a node

Clear speedUse when normal waves are the time cost.
Boss damageUse when a single target blocks progress.
SurvivalUse when the run ends before damage matters.
EconomyUse only when it changes the resource you need next.

Write down the stage, the point at which it fails, and whether the loss is time or survival. That gives a Rune choice a job. A route can look powerful on paper and still be wrong if it does not change the reason the run is stuck.

How to evaluate a Rune route

  1. Name the bottleneck. Is the run too slow, failing a boss, or collapsing late?
  2. Change one meaningful input. Move a Rune point, then use the same target instead of comparing unrelated runs.
  3. Record the result. Clear time, stability, and resource gained per session are more useful than a broad “felt better.”
  4. Keep the patch label. Builds and routes should always state the game version they were tested on.

Rune FAQ

Should I follow one Rune build forever?

No. A practical Rune route is tied to a specific progression goal. Revisit it after a new gear effect, stage wall, or patch changes the result you are chasing.

Where is the Task Bar Hero Rune Tree?

Use the dedicated Rune Tree guide for a goal-first route framework and links back to the broader Rune reference.

Are Rune effects verified here?

Only version-checked data should be treated as confirmed. This first release keeps unverified numeric claims out of the database.
